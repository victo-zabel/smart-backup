# Проект: Скрипт автоматизированного резервного копирования (smart-backup)

## Описание

Bash-скрипт для автоматизации рутинной задачи резервного копирования данных. Скрипт создает сжатый архив (`.tar.gz`) указанной директории, добавляет метку даты к имени файла, удаляет старые бэкапы и отправляет уведомление о статусе в Telegram.

Это мой **первый учебный проект** в рамках освоения базовых навыков Junior DevOps инженера.

## Функциональность

*   Создание архивов `.tar.gz`.
*   Использование динамической даты в имени файла (`backup_YYYY-MM-DD_HH-MM-SS.tar.gz`).
*   Ротация (очистка) старых архивов (удаление файлов старше N дней).
*   Конфигурация через внешний файл (`config.conf`).
*   Уведомления в Telegram через Bot API.

## Используемые технологии

*   **Bash:** Язык скриптинга командной строки Linux.
*   **Утилиты Linux:** `tar`, `find`, `date`, `curl`, `du`.
*   **Git/GitHub:** Контроль версий.

## Использование и конфигурация

### Файл конфигурации (`config.conf`)

Для работы скрипта требуется файл `config.conf` в том же каталоге, где находится сам скрипт. В нем должны быть определены следующие переменные:

```conf
# Путь куда сохранять архивы
BACKUP_DIR="/home/user/Backups"

# Путь к папке, которую нужно архивировать
SOURCE_DIR="/home/user/devops/МойПроект" 

# Сколько дней хранить архивы (например, 7)
KEEP_DAYS=7

# Настройки Telegram API
TELEGRAM_BOT_TOKEN="ВАШ_ТОКЕН"
TELEGRAM_CHAT_ID="ВАШ_CHAT_ID"
 ```

Можно установить атоматизацию через Crontab
Для ежедневного автоматического запуска в 3:00 ночи добавьте следующую строку в crontab -e:

```Bash
0 3 * * * /путь/к/вашему/скрипту/run_backup.sh
```
